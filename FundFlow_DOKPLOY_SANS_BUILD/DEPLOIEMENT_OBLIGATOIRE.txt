FUNDFLOW - DEPLOIEMENT DOKPLOY SANS BUILD

Cette version ne doit PAS être créée comme une "Application" Nixpacks ou Dockerfile.
Elle doit être créée comme un service "Compose".

ETAPE 1 - GITHUB
1. Créez un NOUVEAU dépôt GitHub vide.
2. Décompressez cette archive.
3. Envoyez directement à la racine du dépôt :
   - app.py
   - docker-compose.yml
   - .env.example
   - DEPLOIEMENT_OBLIGATOIRE.txt
4. Vérifiez que docker-compose.yml est visible directement sur la première page du dépôt.
5. N'envoyez pas le fichier ZIP lui-même.

ETAPE 2 - DOKPLOY
1. Dans le projet Dokploy, cliquez sur Create Service / Créer un service.
2. Choisissez COMPOSE, pas Application.
3. Compose Type : Docker Compose, pas Docker Stack.
4. Source : GitHub.
5. Sélectionnez le NOUVEAU dépôt et la branche main.
6. Compose Path : ./docker-compose.yml
7. Enregistrez puis cliquez sur Deploy.

ETAPE 3 - VARIABLES
Dans l'onglet Environment de ce service Compose, ajoutez :
ADMIN_EMAIL=votre-adresse-email
ADMIN_PASSWORD=votre-mot-de-passe-fort

ETAPE 4 - DOMAINE
1. Ouvrez Domains.
2. Ajoutez votre domaine d'application.
3. Service : fundflow
4. Container Port / Target Port : 8000
5. HTTPS : activé seulement lorsque le DNS pointe déjà vers le VPS.

IMPORTANT
- Ne choisissez pas Nixpacks.
- Ne choisissez pas Dockerfile.
- Ne réutilisez pas l'ancienne Application Dokploy.
- Créez un nouveau service Compose pour éviter l'ancienne configuration mise en cache.
- Ne mettez pas admin.gestionflux.fun comme domaine de l'application si ce sous-domaine ouvre déjà le panneau Dokploy.

VERIFICATION
Après le déploiement, ouvrez : https://votre-domaine/health
Réponse attendue : {"ok": true, ...}
